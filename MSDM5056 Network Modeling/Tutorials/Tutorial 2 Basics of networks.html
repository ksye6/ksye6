You can check whether a network is bipartite with is_bipartite. It uses the two-colouring algorithm:

  1. make all nodes uncoloured;
  2. paint a node with one colour (say red);
  3. paint its neigbhours with another colour (say blue);
  4. paint its neighbours' neighbours with the first colour (say red); and
  5. repeat steps 3 and 4.

A network is bipartite if its nodes can be coloured without contradiction, and each set of the coloured nodes forms a partition. (Can you implement this?)

In [3]:

Out[3]:
True

You can split a bipartite network into its two partitions with algorithms.biadjacency.sets. Given a partition, you can then generate the network's biadjacency matrix with algorithms.biadjacency.biadjacency_matrix. Note that it returns a SciPy matrix instead of a NumPy array.

Note that our lecture notes refer to a "biadjacency matrix" as an "incidence matrix", but it means something else in NetworkX.

2. Tree and planarity¶

A tree is an undirected acyclic graph. It has two properties.

  1. An $n$-node tree has exactly $\left(n-1\right)$ edges.
  2. A tree is planar, i.e. it can be drawn on a plane without intersecting edges.

You can easily check whether a network is a tree: just see if it has one more node than its edges. You may also use the method is_tree. In contrast, it is more subtle to check whether a network is planar. You can do this with check_planarity, whose algorithm is based on Kuratowski's theorem.

In [6]:
print('The projection on the blue nodes is %sa tree.' %
      ('' if nx.is_tree(BonB1) else 'not '), end=' ')
print('It is %splanar.' %
      ('' if nx.is_planar(BonB1) else 'not '))

The projection on the blue nodes is not a tree. It is planar.

2.1 Minimum spanning tree¶

A subgraph is a spanning tree if it is a tree and contains all nodes of the network. A spanning tree is a minimum spanning tree (MST) if its edges has the smallest total weight over all spanning trees. A weighted network has a unique MST when its edges carry distinct weights.

You can compute a graph's MST with algorithms.tree.minimum_spanning_tree, which uses Kruskal's algorithm (1956): given a graph $G$,

  1. create an edgeless copy $H$ of $G$;
  2. add (one of) the lightest edge(s) of $G$ to $H$ if it connects two trees;
  3. remove the edge from $G$; and
  4. repeat steps 2 and 3 until $H$ connects all nodes.

Finally, $H$ is the MST of $G$. (Can you implement this?)

In [8]:
for u, v in BonB1.edges: BonB1[u][v]['weight'] = (u*v)/10
weights = [w for u, v, w in BonB1.edges(data='weight')]
MST = nx.algorithms.tree.minimum_spanning_tree(BonB1)

plt.figure(figsize=(7, 2))
plt.subplot(121)
plt.title('projection on the blue nodes\nwith distinct weights\n')
nx.draw(BonB1, pos=BonB1.graph['pos'], with_labels=True,
        node_color='b', font_color='w', edgecolors='k', width=weights)

plt.subplot(122)
plt.title('its minimum spanning tree\n')
nx.draw(MST, pos=BonB1.graph['pos'], with_labels=True,
        node_color='b', font_color='w', edgecolors='k',
        width=[w for u, v, w in MST.edges(data='weight')])
plt.show()

print('\nThe minimum spanning tree is %sa tree.' %
      ('' if nx.is_tree(MST) else 'not '))

The minimum spanning tree is a tree.

3. Degree¶

You can use degree to get a dictionary-like object that contains all nodes' degrees $\left\{k\right\}$. For a DiGraph, degree returns the sum of in-degrees and out-degrees, which can be respectively retrieved with in_degree and out_degree.

karate = nx.karate_club_graph()
for u, v in karate.edges():    # make it unweighted
    del karate[u][v]['weight'] # for simplicity
degrees = [k for n, k in karate.degree]

cmap = mpl.colors.LinearSegmentedColormap.from_list('', ['w', 'r'])
norm = mpl.colors.Normalize(vmin=min(degrees), vmax=max(degrees))
smap = mpl.cm.ScalarMappable(norm, cmap)
smap.set_array([])

There are various ways to plot a network's degree distribution $P_k$. Here I demonstrate three.

A major difference between the ways is about the method of binning. If you do not bin your data, you genuinely show the probability for every degree $k$. Although it is accurate, the resultant plot looks quite clumsy once the network's maximum degree is large.

4. Shortest path¶

The length of the shortest path between two nodes may be referred to as the (geodesic) distance between them, whereas the shortest path may be called the geodesic (line). A network's diameter is the longest distance over all pairs of nodes. In other words, no shortest paths are longer than the diameter. You can measure it with diameter.

NetworkX provides many methods to find shortest paths and compute their lengths. The basic methods of NetworkX, viz. shortest_path and shortest_path_legnth, employ Dijkstra's algorithm (1959). (Can you implement this?)

In [11]:
d = nx.diameter(karate)
print('The diameter of Zachary\'s karate club is %d.' % d)

A = nx.to_numpy_array(karate)
A_power_sum = np.copy(A)
for p in range(2, d):
    A_power_sum += np.linalg.matrix_power(A, p)

# recall that (A**p)[i, j] == 0 implies
# the absence of any p-edge paths between i and j
sources, targets = np.where(A_power_sum == 0)
d_nodes = nx.shortest_path(karate, sources[0], targets[0])
print('One of its longest geodesics is %s.' % d_nodes)

nx.set_edge_attributes(karate, .1, 'geo')
for i in range(d):
    karate[d_nodes[i]][d_nodes[i+1]]['geo'] = 3

plt.figure(figsize=(5, 4))
plt.title('Zachary\'s karate club')
plt.colorbar(smap, fraction=0.045, pad=0.05, label='degree')
nx.draw_kamada_kawai(karate, with_labels=True,
                     edgecolors='k', node_color=degrees, cmap=cmap,
                     width=[w for u, v, w in karate.edges(data='geo')])

The diameter of Zachary's karate club is 5.
One of its longest geodesics is [14, 32, 2, 0, 5, 16].

You can calculate the average distance manually, or you can use average_shortest_path_length.

In [12]:
print('The average distance is %.3g.' %
      np.mean([nx.shortest_path_length(karate, u, v)
               for v in karate for u in karate if u != v])
     ) # equivlaent to nx.average_shortest_path_length(karate)

The average distance is 2.41.

It is sometimes more convenient to use floyd_warshall_numpy, which directly returns a distance matrix between nodes. It employs the Floyd-Warshall algorithm (Floyd, 1962; Warshall, 1962) instead of Dijkstra's algorithm. (Can you implement this?)

You may check Shorteset path problem on Wikipedia to know more about the assumptions and the efficiency of various path-finding algorithms. Choosing an appropriate algorithm can make your life much easier.

5. Component¶

You can check a network's connectedness with is_connected. You should always check it before computing a network's average path length, otherwise errors may arise. In addition, you can break a network into its (connected) components with connected_components and count them with number_connected_components.

For a DiGraph, you must add weakly_ or strongly_ in front of connected in the three methods' names to specify whether you are checking the weakly or the strongly connected components.

In [14]:
toy = nx.read_gml('datasets/toy.gml')
toy = nx.relabel_nodes(toy, {n: int(n) for n in toy})
toy.graph['pos'] = {n: tuple(toy.nodes[n].values()) for n in toy}

print('The toy network from last tutorial is %sconnected.'
      % ('' if nx.is_connected(toy) else 'not '))
print('It has %d connected components.'
      % nx.number_connected_components(toy))

for i, component in enumerate(nx.connected_components(toy)):
    for node in component:
        toy.nodes[node]['c'] = i

plt.figure(figsize=(5, 3))
plt.title('the toy network\'s components\n')
nx.draw(toy, pos=toy.graph['pos'], edgecolors='k', with_labels=True,
        node_color=[c for n, c in toy.nodes(data='c')], cmap='Accent')

The toy network from last tutorial is not connected.
It has 7 connected components.

If a network is disconnected, we are usually interested in its largest component, but NetworkX does not provide a simple method to extract it directly.

In [15]:

A $k$-component is a connected subgraph that remains connected unless you remove $k$ nodes from it. You can get a dictionary of $k$-components keyed with $k$ using algorithms.connectivity.k_components .

nx.draw(lcomp, pos=lcomp.graph['pos'], edgecolors='k',
        node_color=['b' if n in kcomp else 'w' for n in lcomp.nodes])

6. Laplacian matrix¶

A network's Laplacian matrix $\mathbf{L}=\mathbf{D}-\mathbf{A}$ is the difference between its degree matrix $\mathbf{D}$ and adjacency matrix $\mathbf{A}$. You can calculate it manually or obtain it with laplacian_matrix, which returns a SciPy matrix instead of a NumPy array.

In [17]:
np.set_printoptions(threshold=100) # so that L will look smaller

A = nx.to_numpy_array(toy)
D = np.diag(A.sum(0)) # = np.diag([k for n, k in toy.degree])
L = (D-A).astype(int); L

You can get its spectrum (i.e. its set of eigenvalues) with laplacian_spectrum. Spectral graph theory is devoted to study what kind of information is encoded in a network's spectrum. For example, the number of components on a network is found to be equal to the number of zeros in its spectrum.
